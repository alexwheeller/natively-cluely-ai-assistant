# Live Salary Negotiation Coaching — Feature Spec

## What is Natively?

Natively is an Electron desktop app that provides real-time AI assistance during job interviews. The user runs it during a call — it listens to the interviewer via STT (speech-to-text), captures screenshots, and provides live coaching through a chat overlay.

## What already exists

- Resume/JD upload and structured parsing (StructuredResume, StructuredJD)
- Company research dossier (salary estimates, culture ratings, critics, benefits via CompanyResearchEngine + TavilySearchProvider)
- Pre-computed negotiation script (opening_line, justification, counter_offer_fallback, salary_range) generated by NegotiationEngine.ts via AOTPipeline
- IntentClassifier that detects NEGOTIATION intent from keywords: salary, compensation, offer, negotiate, equity, bonus, ctc, expected salary, etc.
- KnowledgeOrchestrator.processQuestion() injects a static `<salary_intelligence>` block when NEGOTIATION intent detected
- SalaryIntelligenceEngine for resume-based salary estimation
- TechnicalDepthScorer for tracking interviewer tone/depth during technical questions
- LLMHelper with streaming chat, Gemini/OpenAI/Claude/Ollama support
- STT: interviewer speech is captured and available in the pipeline

## The Problem

The current negotiation system is completely static:
- It injects the same pre-computed script regardless of what the recruiter actually said
- It does NOT know that the recruiter just offered $95k
- It does NOT know the recruiter pushed back with "that's above our band"
- It does NOT know equity/PTO were mentioned as sweeteners
- Each message is processed in isolation — no negotiation state accumulates
- The user gets generic advice when they need precise, moment-specific coaching

A user in a live salary negotiation call needs:
"They just said $95k. My target is $130k. Tell me exactly what to say right now."
Not: "Here are some general salary negotiation tips."

## Key User Scenarios

1. **Recruiter asks salary expectations cold** → Give user the exact scripted opening line anchored at upper market range
2. **Recruiter drops a specific number below target** (e.g. "budget is $95k") → Detect the offer, calculate gap, suggest precise counter (e.g. "I'm targeting $125k based on X, Y, Z")
3. **Recruiter pushes back** ("that's above our band") → Coach through holding position, referencing market data, or strategic silence
4. **Recruiter pivots to benefits** (equity, signing, PTO, remote) → Help evaluate total comp, suggest how to negotiate each component
5. **User wants to introduce flexibility ask** → Suggest how to bring up remote/hours/title without damaging salary position
6. **Negotiation reaches impasse** → Suggest close tactics or graceful hold ("Can I have 24 hours to consider?")

## What to build

### 1. NegotiationConversationTracker
- Stateful object that accumulates recruiter statements turn by turn
- Extracts signals: offers (dollar amounts), rejections, counteroffers, benefits mentions
- Tracks negotiation phase: PROBE → ANCHOR → COUNTER → HOLD → CLOSE → PIVOT_BENEFITS
- Persists across the full conversation session

### 2. LiveNegotiationAdvisor
- Analyzes the latest recruiter turn in context of full negotiation history
- Detects what just happened (offer dropped, pushback, benefits pivot, etc.)
- Generates a specific, actionable next move for the user
- Produces: what_happened (1 line), recommended_response (exact script), tactical_note (why this move)
- Uses the user's resume achievements + dossier salary data to ground every response in real numbers

### 3. ContextAssembler rules upgrade
- New negotiation-aware rules in the prompt when NegotiationConversationTracker has active state
- Rules for: anchoring, holding, pivoting, total comp evaluation
- Replace "here's your script" with "here's what to say given what they just said"

### 4. UI: Live Negotiation HUD
- Shown in the NativelyInterface overlay when negotiation phase is detected
- Displays: current phase badge, latest tactical recommendation, salary range reference
- Non-intrusive but immediately visible — user glances at it during the call
- Shows the recruiter's detected offer vs user's target vs market range
- Quick-copy buttons for the recommended response

### 5. Integration into KnowledgeOrchestrator
- NegotiationConversationTracker is instantiated per session
- processQuestion() feeds recruiter speech into tracker before generating response
- LiveNegotiationAdvisor replaces static salary_intelligence injection when tracker has state
- Tracker state resets when JD is changed or session ends

## Technical Constraints

- Electron + React, TypeScript throughout
- IPC via safeHandle pattern
- Must integrate into existing KnowledgeOrchestrator.processQuestion() pipeline
- Must work with all LLM providers (Gemini, OpenAI, Claude, Ollama) via generateContentFn
- No new external dependencies if possible
- Performance: live coaching must respond within the same latency budget as existing chat (~2-4s)
- The STT interviewer speech is currently fed via `feedInterviewerUtterance()` on KnowledgeOrchestrator

## Success Criteria

The feature is a success if:
- A user who just heard "our budget is $95k" gets a specific counter-offer script with their actual target number within 3 seconds of asking
- The system tracks at least: last offer, pushback count, benefits mentioned
- Phase transitions are accurate enough to give phase-appropriate advice
- The UI shows the negotiation state at a glance without requiring the user to scroll or click
